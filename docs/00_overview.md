# LG Group AI Strategy Wiki Overview

이 Wiki는 공개 출처를 바탕으로 LG그룹의 AI 사업기회, 개념 체계, 실행 주체, 전략 질문을 연결한다. Docs와 Topics가 원본 지식의 중심이며 Concepts, Companies, Sources는 이를 기반으로 정리된 파생 Entity다.

## 1. Six Themes

| No. | Theme | Strategic Focus | Document |
|---:|---|---|---|
| 01 | AI Data Center / Infra | GPU보다 전력·냉각·운영 효율까지 포함한 One LG AIDC stack | [[docs/01_ai_data_center_infra]] |
| 02 | Physical AI / Smart Manufacturing | 제조 데이터·로봇·디지털트윈을 AI Factory 운영 모델로 연결 | [[docs/02_physical_ai_smart_manufacturing]] |
| 03 | AI Mobility / SDV·AIDV | 완성차 없이 display·sensing·battery SW·connectivity 레이어 확보 | [[docs/03_ai_mobility_sdv_aidv]] |
| 04 | Enterprise AX / Agentic Operating Model | 전사 ontology와 Agent workflow 기반 운영체계 재설계 | [[docs/04_enterprise_ax_agentic_operating_model]] |
| 05 | AI for Science / Bio / Materials / Battery | AI 후보 제안과 실험·검증을 연결하는 closed-loop R&D | [[docs/05_ai_for_science_bio_materials_battery]] |
| 06 | Global AI Alliance / Open Innovation | 외부 기술을 빠르게 활용하면서 데이터·운영 지식을 내부화 | [[docs/06_global_ai_alliance_open_innovation]] |

`AI Factory`는 Theme 1이 아니라 Theme 2의 핵심 Concept이다. AI Data Center는 기반이고 AI Factory는 그 위에서 Physical AI를 학습·검증·배포하는 운영 모델이다.

## 3. Entity Relationships

```text
Theme Docs ─┬─ Strategic Topics
            ├─ Concepts
            ├─ LG Affiliates / Partners
            ├─ Source Cards
            └─ Images / Structured Data
```

- Concepts는 반복 등장하는 개념의 정의와 경계를 관리한다.
- Companies는 이 Wiki에서 확인된 전략적 역할만 정리한다.
- Sources는 원문 사실과 reliability를 관리하며 전략 해석은 Docs와 Topics에 둔다.
- Assets는 `assets/images/` 단일 폴더를 사용하고 재사용 관계는 inventory로 관리한다.
